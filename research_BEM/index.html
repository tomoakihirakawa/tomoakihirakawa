    多重極展開とその移動¶

    sh clean
cmake -DCMAKE_BUILD_TYPE=Release ../ -DSOURCE_FILE=test_translation_of_a_multipole_expansion.cpp
make
./test_translation_of_a_multipole_expansion
    ここで示す，多重極展開は次式を近似する．

    \[\begin{equation} (G,\nabla G\cdot {\bf n})=\left(\frac{1}{\|{\bf x}-{\bf a}\|}, -\frac{{\bf x}-{\bf a}}{\|{\bf x}-{\bf a}\|^3}\cdot{\bf n}\right) \end{equation}\]

    ガウス・ルジャンドル積分を使う際には，これに重みをかけて足し合わせる．その重みは別に計算し，保存しておく．元々の関数の近似の係数と重みの係数を混同しないように注意する．現在のところ，以下のような値を与えて多重極展開を計算している．

    • カーネル\(G\)には，位置と数値積分のための重みを与えている．
    • カーネル\(\nabla G\cdot {\bf n}\)には，位置と数値積分のための重み，そして法線ベクトルを与えている．
    void increment(const Tddd& XIN, const std::array<double, 2> weights, const Tddd& normal) {
   const Tddd R = XIN - this->X;
   auto set_coeffs = [&](int n, int m) -> std::array<std::complex<double>, 2> {
      return {SolidHarmonicR(n, m, R) * weights[0],
               Dot(normal, Grad_SolidHarmonicR(n, m, R)) * weights[1]};
   };
   this->set(set_coeffs);
};
    \[\begin{align*} \begin{pmatrix} G\\ \nabla G\cdot {\bf n} \end{pmatrix} & = \begin{pmatrix} \frac{1}{\|{\bf x}-{\bf a}\|}\\ -\frac{{\bf x}-{\bf a}}{\|{\bf x}-{\bf a}\|^3}\cdot{\bf n} \end{pmatrix} \\ & \approx \sum_{n=0}^{N}\sum_{m=-n}^{n} \begin{pmatrix} \overline{S}_{n,m}R_{n,m}\\ \overline{S}_{n,m}((\nabla_{\rm sphe}R_{n,m})J)\cdot{\bf n} \end{pmatrix}\\ & = \sum_{n=0}^{N}\sum_{m=-n}^{n} \begin{pmatrix} \overline{S}_{n,m}R_{n,m}\\ \overline{S}_{n,m}F_{n,m} \end{pmatrix} \end{align*}\]

    \(J\)は，極座標を直交座標で微分したものであり，ヤコビアン行列である．なので，極座標による微分に\(J\)をかけることで，直交座標での微分を得ることができる．なので，\(((\nabla_{\rm sphe}R_{n,m})J)\)は\(\nabla R_{n,m}\)と同じである．

    \(\overline{S}_{n,m}=\overline{S}_{n,m}({\bf a},{\bf x}_c)\),\(R_{n,m} = R_{n,m}({\bf x},{\bf x}_c)\),\(F_{n,m} = F_{n,m}({\bf x},{\bf x}_c)=\nabla R_{n,m}({\bf x},{\bf x}_c)\cdot{\bf n}({\bf x})\)である．

    モーメント¶

    多重極展開におけるモーメントは，簡単に言えば，\(R_{n,m}\)や\(F_{n,m}\)に，チャージまたはポテンシャルなどの値をかけたものの積分である．数値的には，\(R_{n,m}\)や\(F_{n,m}\)に，チャージまたはポテンシャルなどの値をかけたものに数値積分の重みをかけて足し合わせたものである．

    GMRESと高速多重極展開¶

    GMRES法は線形連立方程式を解くための反復法である．GMRESには行列ベクトル積の計算が含まれており，GMRESにとって行列ベクトル積の計算が最もコストがかかる．

    高速多重極展開は，多重極展開とその展開中心の移動，ツリー構造によって，この行列ベクトル積の計算を高速に行うことができる．

    具体的な計算をまとめてみよう．BEMに対してGMRES法を使う際に，行列ベクトル積の，行列とは何か，またベクトルとは何か？行列は，\(G\)や\(\nabla G\cdot {\bf n}\)の境界面積分である．そうなると，ベクトルは速度ポテンシャルと思われるが，GMRES法においては，クリロフ部分空間の正規直交基底\(\{v_1,v_2,\cdots,v_m\}\)がベクトルとして与えられる．この基底を使って，何度も行列ベクトル積を計算し足し合わせたものが，解の近似値となる．

    • GMRESは，繰り返し，異なる基底ベクトルを与え，行列ベクトル積を計算する

    その際に，高速多重極展開はどのような計算を行い，どのように行列ベクトル積を計算するのか？

    基底ベクトルが与えられた，FMMクラス（仮に）は，基底ベクトルからモーメントを計算する． 具体的に，\(R_{n,m}\)や\(F_{n,m}\)に，基底ベクトルをかけたものに数値積分の重みをかけたものを足し合わせる． 違う基底ベクトルに対しても，\(R_{n,m}\)や\(F_{n,m}\)と数値積分のための重みは共通なので，それらを保存しておけば，計算量を削減できる． このように考えると，プログラムは，

    1. FMMクラスの初期化：

      1. \(R_{n,m}\)や\(F_{n,m}\)と数値積分のための重みを計算し配列に保存
      2. 展開中心の移動に必要な係数を配列に保存

    2. （行列ベクトル積）GRMESから基底ベクトルを与える：

      1. 保存しておいた\(R_{n,m}\)や\(F_{n,m}\)と数値積分のための重みを使って，モーメントを計算
      2. 保存しておいた展開中心の移動に必要な係数を使って，展開中心を移動

    行列ベクトル積の結果ベクトルをGMRESに与える．2.を繰り返す．
